GitHub Documentation


1. git init

It is used to initialize a empty git repository in your specified folder (project folder)

2. git status

. git status command shows the current condition of the working directory and staging area.

. It shows you which modifications have been staged and that have not yet, as well as which files are not being monitored by Git.

. The status output provides no information about the committed project history.

3. git status --short

Short status flags are:

  • ?? - Untracked files
  • A - Files added to stage
  • M - Modified files
  • D - Deleted files

4. git add

  • The git add command adds a change in the working directory to the staging area.
  • It notifies Git that you wish to include changes to a certain file in the next commit.
  • git add has no effect on the repository since changes are not truly recorded until you execute git commit.

4.1 git add .

Add all the files of the current working directory

5. git rm

  • The git rm command removes a file or group of files from a Git repository.
  • A file is removed from both your machine and the Git repository. To preserve the file on your local machine, use the –cached flag.

5.1 git rm –cached

  • The Git rm –cached flag removes a file from the staging area.
  • The files from the working directory will remain intact. This means that you’ll still have a copy of the file locally. The file will be removed from the index tracking your Git project.

6. git clone

  • Git clone is generally used to point to an existing repository and create a clone or duplicate of that repository in a different directory.
  • The original repository might be on the local filesystem or on a remote computer that supports the relevant protocols.
  • The git clone command duplicates an already existing Git repository.

7. git commit -m

  • A commit is an operation which sends the latest changes of the source code to the repository, making these changes part of the head revision of the repository.
  • When we commit, we should always include a message.
  • By adding clear messages to each commit, it is easy for yourself (and others) to see what has changed and when
  • It is possible to commit changes directly, skipping the staging environment. The -a option will automatically stage every changed, already tracked file.

8. git Push

  • The git push command is used to upload local repository content to a remote repository. Pushing is how you transfer commits from your local repository to a remote repo
  • Establish A link between remote and local by using git remote add origin link

9. git Pull

  • The git pull command is used to fetch and download content from a remote repository and immediately update the local repository to match that content. Merging remote upstream changes into your local repository is a common task in Git-based collaboration work flows.

11. git branch

  • The git branch command allows you to create, list, rename and delete branches. Many operations on branches are applied by git checkout and git merge command.
  • This command show the current branch in which you are working

12. git checkout

  • The git checkout command lets you navigate between the branches created by git branch .
  • Checking out a branch updates the files in the working directory to match the version stored in that branch, and it tells Git to record all new commits on that branch.
